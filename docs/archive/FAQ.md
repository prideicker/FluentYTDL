# FluentYTDL 常见问题解答 (FAQ)

> [!TIP]
> FluentYTDL 内置了强大的**自动化错误检测与修复引导**。遇到问题时，应用会自动弹出诊断面板帮您定位根因并一键修复。请优先信任应用内的自动引导。
>

## 🚀 快速入门

### 需要自己安装依赖组件吗？

**不需要。** 以下所有核心依赖都会在**首次启动时自动下载安装**，您只需保持网络畅通：

| 组件 | 用途 |
|------|------|
| **yt-dlp** | 核心下载引擎，解析链接、获取流媒体数据 |
| **FFmpeg** | 合并音视频轨、转码封装、嵌入封面与字幕 |
| **Deno** (JS Runtime) | 解析 YouTube 加密签名，缺失时可能丢失大量格式 |
| **POT Provider** | 动态生成 PO Token，绕过机器人检测 |
| **AtomicParsley** | 向 MP4/M4A 文件嵌入封面和元数据标签 |

在 **设置 > 核心组件** 页面可查看安装状态、当前版本，并一键检查更新。

> 📖 深入了解：[组件体系与更新渠道](./ADVANCED_GUIDE.md#-6-组件体系与更新渠道)

### 必须登录 YouTube 吗？

**绝大多数公开视频不需要登录。** 只有以下场景需要导入 Cookie：
- 年龄限制（18禁）内容
- 频道会员专属内容
- 受强力风控保护的极高码率流（部分场景）

日常使用直接复制链接粘贴解析即可。

### 需要特殊网络环境吗？

**是的。** FluentYTDL 必须能连接 YouTube 国际服务器。中国大陆等受限网络环境须配置代理工具，详见 [网络与代理](#-网络与代理) 章节。

### 支持 B 站 / Patreon 吗？

**可以。** 底层引擎 yt-dlp 支持数百个网站。但 FluentYTDL 的 UI 和防封控优化**专为 YouTube 定制**，其他站点可能需手动提供 Cookie，兼容性不做 100% 担保。

### 支持 iOS 吗？

**不支持。** 受限于 iOS 封闭的文件管理和后台机制，暂无 iOS 版本计划。

### 如何默认下载 MP3？

在下载配置面板中，将"下载策略"切换为 **"自定义"** → 选择"仅提取音频" → 指定格式为 `MP3`。

## 🔐 Cookie 与身份验证

### 推荐哪种 Cookie 获取方式？

FluentYTDL 提供三种方式，**推荐程度从高到低**：

| 方式 | 推荐度 | 一句话说明 |
|------|--------|-----------|
| **DLE 登录获取** | ⭐⭐⭐ 强烈推荐 | 弹出安全沙盒窗口登录，一键获取，无兼容性问题 |
| **Firefox 自动提取** | ⭐⭐ 推荐 | 无需管理员权限，稳定可靠 |
| **手动导入** | ⭐ 兜底 | 用扩展导出 cookies.txt 后导入，需定期更新 |

> [!WARNING]
> **Chrome 自动提取极不稳定**（即使管理员权限也常失败），**Edge 必须管理员权限**。如果你正在使用这两个浏览器提取 Cookie 且频繁失败，请切换到 DLE 或 Firefox。

> 📖 深入了解：[Cookie 全流程解析](./ADVANCED_GUIDE.md#-3-cookie-全流程解析)

### 提示 "Sign in to confirm you're not a bot" 怎么办？

按优先级操作：
1. **点击应用弹出的自动修复引导** — 自动诊断根因并一键处理
2. **更新 yt-dlp** — `设置 > 核心组件` 一键更新（最有效）
3. **刷新 Cookie** — 推荐 DLE 重新登录，或切换到 Firefox 提取
4. **更换代理节点** — 当前 IP 可能已被标记

### 浏览器无法自动获取 Cookie？

**原因**：Chrome/Edge 等 Chromium 浏览器自 v127 起引入 App-Bound Encryption，禁止第三方读取 Cookie。

**解决**：切换到 **Firefox** 提取，或使用 **DLE 登录获取**（完全绕过此限制）。

### Cookie 检测不通过？

常见原因：Cookie 过期、浏览器已登出、修改了账号资料触发风控、导出格式不规范。

**解决**：使用 DLE 重新登录获取，或在浏览器重新登录后再次提取。

### 显示"没有 Cookie"怎么办？

不影响下载公开视频。只有权限相关的下载阻断时才需补充 Cookie，平时可以无视。

### 改了用户名/头像后出现验证错误？

修改账号资料触发了 Google 安全风控。**解决**：浏览器退出 YouTube → 重新登录 → 播放一个视频确认正常 → 重新获取 Cookie。

## 🌐 网络与代理

### 网络正常但下载失败？

浏览器能看视频≠软件能下载。代理软件可能只接管了浏览器流量。

**解决**：在代理软件中开启 **系统代理** 或 **TUN 模式**：
- **V2RayN**：底部切换为「系统代理」，或开启 TUN
- **Clash Verge / CFW**：启用「System Proxy」开关，不行就开 TUN
- **其他**：确保开启"系统代理"/"全局模式"，非仅"规则/PAC 模式"

FluentYTDL 默认"跟随系统"代理，开启系统代理后无需额外配置。

### 代理开启后仍失败 / DNS 错误？

1. **端口未对齐**：在 `设置 > 网络连接` 改为"手动配置"，填入代理端口（如 `http://127.0.0.1:7890`）
2. **节点被封锁**：更换其他国家/地区的节点

> [!IMPORTANT]
> 不推荐长期使用手动代理。代理软件换端口后需手动同步修改，容易遗漏。

### HTTP 403 / 429 错误？

**403（拒绝访问）**：优先更新 yt-dlp；仍不行则更换代理节点（IP 封锁需等 12-24 小时解封）。

**429（限流）**：暂停 2-12 小时后重试，或重启路由器 / 换代理节点获取新 IP。

### 下载速度忽快忽慢？

主要原因：YouTube 服务器端动态限速、代理带宽波动、分片下载阶段性差异。

**改善**：确保 Deno 已安装 → 更新 yt-dlp → 换更稳定的代理节点 → 调整并发分片数。

> 📖 深入了解：[性能调优与并发控制](./ADVANCED_GUIDE.md#-5-性能调优与并发控制)

## 📥 下载与格式

### 为什么最高只有 1080p，网页上看有 4K/8K？

常见原因：
1. **视频刚发布** — 4K/8K 编码需数小时，稍后重试
2. **风控降级** — 系统回退到 1080p，需导入 Cookie + 换干净 IP
3. **缺少 Deno** — JS 运行时缺失导致格式丢失
4. **yt-dlp 过期** — 更新组件

FluentYTDL 解析后会在日志显示最高可用分辨率（如 `🎉 最高可用分辨率: 4320p (8K)`），可据此判断。

> 📖 深入了解：[客户端策略与画质原理](./ADVANCED_GUIDE.md#-1-youtube-客户端策略与画质原理)

### 自动画质异常？

除上述原因外：POT Provider 未运行、Cookie 账号等级不足（非 Premium）、或系统自动选了兼容格式。可在下载面板手动指定分辨率。

### 提示"无可用视频流"？

- 视频有年龄/会员限制但未导入 Cookie
- 视频已删除、私享或锁区
- yt-dlp 或 Deno 需更新

### MP4 vs 原盘格式？

**MP4**：兼容性最佳，但 4K+ 可能需重编码。**原盘（MKV/WebM）**：画质最优、合并最快，但老播放器可能不支持。

> 📖 深入了解：[视频格式、编码与容器兼容性](./ADVANCED_GUIDE.md#-7-视频格式编码与容器兼容性) — 为什么有些视频在手机/电视上无法播放

### DRM 保护内容无法下载

YouTube Movies & TV 的付费内容受 **Widevine DRM** 强加密保护。

> [!CAUTION]
> FluentYTDL 及市面上绝大多数工具**不支持也无法解密 DRM 内容**。付费电影/剧集、YouTube Originals、部分音乐 MV 完整版均属此类。遇到报错是正常行为，非软件 Bug。

### 直播无法从开头开始？

技术限制：只能从当前直播时间点开始录制。建议等直播结束后下载完整录播存档（VOD）。

### VR 视频下载说明

可正常下载。内置 `android_vr` 客户端可获取最高 8K VR 格式。画面在普通播放器中呈畸变是正常的，需用 VR 头显或全景播放器（Skybox、PotPlayer 全景模式）。

> 📖 深入了解：[VR 视频技术详解](./ADVANCED_GUIDE.md#-4-vr-视频技术详解)

### 播放列表中隐藏视频被解析？

正常。底层元数据包含的视频都会被抓取。不需要的可在多选列表中取消勾选。

### 下载多了会导致账号被封吗？

**极小概率。** 普通日常下载不会封号。但短时间极高并发批量下载数千视频可能触发 429 限流，导致 IP 被暂时封锁 12-24 小时或频繁弹验证。保持默认并发设置即可。

## ❌ 错误与排障

### 软件"针对我"，想用的时候就不行？

不是 😉 YouTube 频繁修改反爬规则，变动时全球下载工具集体失效。**解决**：`设置 > 核心组件` 更新 yt-dlp，99% 的问题更新即解。

### 其他工具能下但 FluentYTDL 不行？

不同工具使用不同 API 通道，YouTube 风控分批次封锁。**解决**：更新 yt-dlp + 刷新 Cookie。

### 杀毒软件误报？

**典型误报。** PyInstaller 打包的无签名程序常被 360 等拦截。代码完全开源，请将安装目录加入白名单。

### 和 ytdlnis / IDM / 油猴的对比

**ytdlnis**：同样基于 yt-dlp，核心算法不更新同样会失效。FluentYTDL 内置自动组件更新，无需手动找版本。

**IDM / 油猴脚本**：依赖脆弱的直接链接嗅探，YouTube 改签名算法或加 PO Token 后立刻失效。FluentYTDL 内置 PO Token 生成引擎和完整的客户端握手模拟，抗风控能力更强。

## 📋 日志与反馈

### 如何获取运行日志？

1. 进入 **设置页** → **系统** 分组 → **日志管理** 卡片
2. 点击右侧 **📁 文件夹图标** 打开日志目录
3. 找到当天 `.log` 文件（如 `app_2026-04-23.log`），复制发送给开发者或附在 GitHub Issue 中

> [!TIP]
> 也可点击"查看日志"在应用内实时预览，支持级别过滤和关键词搜索。
