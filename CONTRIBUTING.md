Thank you for contributing to FluentYTDL! Please follow these guidelines:

- Keep changes small and focused; include tests for new behavior.
- Run linters and tests locally before submitting changes.
- Provide a clear description explaining the motivation and changes.

Code style:
- Follow existing project style (PEP8-ish). Use `ruff` for linting.

Security & secrets:
- Never commit `config.json`, credentials, API tokens, or cookies. Use `config.example.json`.

License:
- By contributing, you agree to license your contribution under the project's MIT license.
